*REST API

    GET -> Get some post
    POST -> Create
    PUT -> Update
    DELETE -> Delete

*HTTP | Hypertext Transfer Protocol

    1xx - informational

    2xx - success
    200 -> Everything worked normal
    201 -> new resouce successfully created

    3xx - redirection
    301 -> client site changes to another website

    4xx - client errors
    400 -> Bad request
    401 -> unauthorized
    403 -> Forbidden
    404 -> Not found
    429 -> too many request | use rate limiter to prevent abuse

    5xx - server errors
    500 -> internal errors
    503 -> service unavalaible (temporary)

*Database -> No SQL & SQL

    SQL
        Structured data 
        SQL Language (SELECT INSERT FROM WHERE)
        Best for complex and strict queries - Bank app, Social media

    No SQL 
        Flexible data
        Query Language or API
        Best for big data and real time app - Game, Streaming
